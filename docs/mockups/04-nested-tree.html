Nested tree navigation

A single expandable tree is the primary navigator: repository → worktree → session. Selecting a node updates the main pane. Click chevrons to expand/collapse. Useful when users think in “where am I in the hierarchy” rather than “pick a repo, then a table row.”

Strengths

  • One navigation surface — no competing “repo list” vs “worktree table”
  • Context path (repo → worktree → session) is always visible
  • Powerful for users who hop between nested items often
  • Familiar pattern (IDE explorer / file tree)

Trade-offs

  • Deep trees get noisy with many repos and worktrees
  • Table-style scanning of worktree status is weaker
  • Selection state across three levels needs careful design
  • Less approachable for first-time users than a flat overview
