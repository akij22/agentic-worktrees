Discussion notes

Shared product direction (unchanged)

  • Repository is the primary entry point
  • Worktrees and sessions stay repository-scoped
  • Dense, operational UI with warm accent
  • Only existing user-facing capabilities illustrated

What these alternatives probe

  • How always-visible should the repo list be?
  • Is hierarchy (tree) better than master-detail?
  • Can keyboard-first switching replace persistent lists?
  • How does the layout behave when a coding session needs width?
